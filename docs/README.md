# PersonalPortal

A self-hosted personal activity portal designed for Dreamhost shared hosting.
Built with PHP 8+, MySQL, and vanilla JavaScript — no frameworks or build tools required.

---

## Feature Overview

| Widget | Description |
|--------|-------------|
| **Bookmarks** | Grouped by category, favicons auto-fetched, click-through links |
| **Clock/Date** | Live digital clock in the header, updates every second |
| **Stock Ticker** | Scrolling ticker tape + detailed widget; refreshes every 5 min |
| **News** | Aggregated RSS feeds, sorted by recency; refreshes every 10 min |
| **Notes** | Bullet-formatted notes with headings; edit via Admin |
| **Admin Panel** | Password-protected CRUD for all content |
| **MFA / 2FA** | TOTP two-factor auth (Authy, Google Authenticator, 1Password, etc.) |

## Installation on Dreamhost

### 1. Create a MySQL Database

1. Log into the [Dreamhost Panel](https://panel.dreamhost.com)
2. Go to **Databases → MySQL Databases**
3. Create a new database (e.g. `yourusername_portal`)
4. Create a database user and assign full privileges
5. Note the **hostname** shown (e.g. `mysql.yourdomain.com`)

### 2. Upload Files

**Option A — Git (recommended)**
```bash
# SSH into your Dreamhost server
ssh user@yourdomain.com

# Navigate to web root or a subdirectory
cd ~/yourdomain.com

# Clone the repo
git clone <repo-url> portal
cd portal
```

**Option B — FTP/SFTP**
Upload all files to a directory under your domain's web root
(e.g. `~/yourdomain.com/portal/`).

### 3. Set Directory Permissions

```bash
chmod 755 cache/
chmod 644 .htaccess
```

### 4. Run the Installer

Open your browser and navigate to:
```
https://yourdomain.com/portal/install/
```

The wizard will:
- Test your database connection
- Import the schema and seed data
- Write `config/config.php`
- Create an admin account

### 5. Delete the Install Directory

**This is important!** After successful installation:
```bash
rm -rf install/
```
Or rename it: `mv install install_bak`

### 6. Log Into Admin

```
https://yourdomain.com/portal/admin/
```

Use the username/password you set during installation.

## Configuration

After installation, `config/config.php` contains all settings:

```php
define('DB_HOST',    'mysql.yourdomain.com');
define('DB_NAME',    'yourdatabase');
define('DB_USER',    'dbuser');
define('DB_PASS',    'dbpass');
define('APP_NAME',   'Personal Portal');  // Title shown in header
define('APP_URL',    'https://yourdomain.com/portal');
define('TIMEZONE',   'America/New_York'); // PHP timezone string
define('STOCK_CACHE_TTL',  300);  // seconds between stock refreshes
define('NEWS_CACHE_TTL',   600);  // seconds between news refreshes
```

To change the timezone, use any value from [PHP's timezone list](https://www.php.net/manual/en/timezones.php).

## Usage Guide

### Adding Bookmarks

1. Go to **Admin → Categories** and create at least one category
2. Go to **Admin → Bookmarks → Add Bookmark**
3. Fill in Title, URL, Category; optionally add a Description (shown as tooltip)
4. Use **Sort Order** to control display order within a category (lower = first)

### Notes Format

Notes support simple markup:

```
## This becomes a heading

- This is a bullet point
* This is also a bullet point

Plain text becomes a paragraph.
```

Edit notes at **Admin → Notes**. The editor includes a live preview pane.

### Managing Stock Symbols

Go to **Admin → Settings → Stock Symbols**. Enter one symbol per line:

```
AAPL|Apple
MSFT|Microsoft
SPY|S&P 500 ETF
QQQ
^DJI|Dow Jones
```

The optional `|Label` after the symbol overrides the display name.
Symbols use Yahoo Finance's public API — most NYSE/NASDAQ/ETF/index symbols work.

> **Note:** The Yahoo Finance endpoint is unofficial. If quotes stop loading, check the browser console for API errors. An Alpha Vantage fallback key can be set in `config.php` (`ALPHAVANTAGE_KEY`).

### Managing News Feeds

Go to **Admin → Settings → News Feeds**:
- **Add Feed** — enter any RSS 2.0 or Atom feed URL
- Toggle feeds ON/OFF without deleting them
- Remove feeds with the ✕ button

Feeds are cached for `NEWS_CACHE_TTL` seconds (default 10 minutes).
Cache is automatically cleared when you add/remove/toggle a feed.

**Suggested feeds:**

## Security Notes

- The admin panel is protected by session-based authentication with CSRF tokens
- Passwords are stored as bcrypt hashes (never plain text)
- All database queries use PDO prepared statements (SQL injection safe)
- The `config/`, `includes/`, and `cache/` directories are blocked by `.htaccess`
- Session cookies are `httponly` and `samesite=Lax`; set `secure=true` (enabled by default) for HTTPS hosts
- The installation wizard locks itself after first run (`install/installed.lock`)

**Recommended:** Add HTTP Basic Auth to the `/admin/` path via Dreamhost panel for an extra layer of protection.

## Two-Factor Authentication (MFA)

PersonalPortal supports TOTP-based MFA compatible with **Authy**, Google Authenticator, 1Password, Bitwarden, and any RFC 6238 app.

### Enabling MFA

1. Log into the admin panel
2. Go to **Admin → Two-Factor Auth**
3. Install Authy (or another authenticator app) on your phone
4. Scan the QR code shown, or enter the manual key into Authy
5. Enter the 6-digit code your app shows to confirm — MFA is now active

### Login flow with MFA enabled

```
Enter username + password → Verify 6-digit TOTP code → Access granted
```

The pending state (after password, before TOTP) expires after **5 minutes**.

### Disabling MFA

Go to **Admin → Two-Factor Auth** and enter your current password to disable.
This removes the stored secret from the database.

### Security details

- Secrets are stored **AES-256-CBC encrypted** in the database, keyed to `SECRET_KEY` in `config/config.php`. A database dump alone cannot clone your TOTP device — the attacker also needs the config file.
- TOTP verification accepts the current 30-second window ±1 (90-second tolerance) to handle clock drift.
- The TOTP verification page shares the same IP-based rate limiter as the login page: 5 failures triggers a 15-minute lockout.
- The QR code is generated **client-side** using qrcodejs — the secret is never sent to any third-party server.

### Lost access / locked out

If you lose your authenticator device:

1. SSH into your server
2. Run the following MySQL command to disable MFA directly:
   ```sql
   UPDATE portal_settings SET setting_value = '0' WHERE setting_key = 'totp_enabled';
   UPDATE portal_settings SET setting_value = '' WHERE setting_key = 'totp_secret';
   ```
3. Log in with just your password, then re-enroll a new device.

## Troubleshooting

| Problem | Solution |
|---------|----------|
| Blank white page | Enable PHP errors temporarily: add `ini_set('display_errors',1);` at top of `index.php` |
| "Database connection failed" | Check `config/config.php` credentials; verify the MySQL host from Dreamhost panel |
| Stock data not loading | Open browser DevTools → Network tab; check `api/stocks.php` response |
| News not loading | Verify RSS feed URLs are reachable from server: `curl -I <feed_url>` via SSH |
| Admin login not working | Ensure cookies are enabled; check `SESSION_NAME` in config; re-run install if needed |
| Cache stale | Delete `cache/*.cache` files or wait for TTL to expire |
| 403 on /admin/ | Verify `.htaccess` doesn't conflict with Dreamhost's own `.htaccess` rules |
